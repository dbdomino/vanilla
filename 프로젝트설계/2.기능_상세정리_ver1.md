# 2.기능 상세정리
## 개요
``` 요구사항을 만족하기 위한 기능을 상세하게 설계한다. ```
1. 가계부를 작성하는 기능
2. 지출을 관리하는 기능 - 고정지출
3. 지출을 관리하는 기능 - 일반지출
4. 통계 처리기능 - 배치 프로세스
5. 회원 관리 기능
### 1. 가계부를 작성하는 기능
- **1.1 월 기준으로 가계부 리스트 작성**
  - 출력 필수조건(연, 월)
  - 날짜 / 사용금액 / 분류 / 사용내역 / 장소 / 수단
  - 출력(전체, 분류, 수단, 날짜), 입력, 수정, 삭제
  - 정렬 기준, 날짜 1순위, 분류 2순위, 수단 3순위
  - 합계(금액 합계)
  - 테이블 위에 한줄씩 입력하는 form으로 작성
  - 입력시 데이터 타입 검증 수행
  - 테이블에 행 마다 기능추가
    - 신규 입력시 행 우측에 ✍️(등록) 🗙(취소) 기능 추가
    - 생성된 단일 행 우측에 ✏️(수정) 🚮(삭제) 기능 추가
    - 생성될 때 식별자 부여됨, 입력완료 후 입력컬럼 삭제, 및 조회 테이블에 추가
  - 기본값(날짜 : 오늘)(사용금액 : 필수)(분류 : '--')(사용내역 : '')(장소 : '')(수단: '현금')
- **1.2 이후 추가 개발 가능성**
  - 가계부 정렬기능 
    - (날짜기준 오름차순 내림차순 바꿔서 정렬 변경)
    - (분류기준, 카드기준 정렬)
  - 가계부 조건 검색기능 (날짜 기준으로 검색)
  - 저장 버튼으로 여러 컬럼을 한번에 수정할 수 있는 기능 고려
  - 저장 버튼으로 작성중인 모든 내용 저장 기능 고려
  - 여러 컬럼을 한번에 삭제하는 기능 고려
  - 고정지출 기능을 매달 테이블에 고정적으로 출력 시킬지 고민
### 2. 지출을 관리하는 기능 - 고정지출
- **2-1 고정지출 테이블 생성**
  - 매월 고정으로 나가는 금액을 가계부에 입력하지 않고 별도 관리
  - 지출이름 / 금액 / 날짜 / 지출방법 / 분류 / 그룹 컬럼
  - 그룹은 1(기본), 2, 3, 4, 5 로 구분
  - 그룹별 합계, 총 합계 출력
  - 입력, 조회(전체), 수정, 삭제 기능
  - 입력시 데이터 타입 검증 수행
- **2-2 이후 추가개발 가능성**
  - 그룹을 커스텀하게 관리하는 기능
### 3. 지출을 관리하는 기능 - 일반지출을 분류로 구분
- 3-1 분류 테이블 관리
  - 가계부에서 지출에 등록되는 분류를 정의, 분류별로 매달 한도 금액을 설정
  - 분류 / 금액 / 날짜 / 한도금액 / 이번달 사용율 
  - 분류 총합계, 이번달 사용금액 합계, 퍼센티지로 비교
  - 분류 리스트를 가계부에서 선택해서 쓰는데, 분류명이 수정된다면 가계부에도 적용 되어야 한다.
  - 분류명이 삭제된다면 가계부에는 분류정보가 '--'로 표시되어야 한다. (빈 분류로 표시됨.)
  - 
- 3-2 **이후 추가개발 가능성**
  - 한도금액 몇퍼 이상이면 색깔 바꾸게 하는 기능
