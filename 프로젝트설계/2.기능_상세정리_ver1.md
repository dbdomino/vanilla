# 2.기능 상세정리
## 개요
``` 요구사항을 만족하기 위한 기능을 상세하게 설계한다. ```
1. 가계부를 작성하는 기능
2. 지출을 관리하는 기능 - 고정지출
3. 지출을 관리하는 기능 - 일반지출
4. 통계 처리기능 - 배치 프로세스
5. 회원 관리 기능
### 1. 가계부를 작성하는 기능
- 월 기준으로 가계부 리스트 작성
#### &nbsp;&nbsp;&nbsp;&nbsp;1.1 가계부 지출 등록 (일시불, 할부)
#### &nbsp;&nbsp;&nbsp;&nbsp;1.2 가계부 지출 조회(단건)
#### &nbsp;&nbsp;&nbsp;&nbsp;1.3 가계부 지출 수정 (일시불, 할부)
#### &nbsp;&nbsp;&nbsp;&nbsp;1.4 가계부 지출 삭제
#### &nbsp;&nbsp;&nbsp;&nbsp;1.5 가계부 목록 조회 (월단위)
  - 출력 필수조건(연, 월)
  - 날짜 / 사용금액 / 분류 / 사용내역 / 장소 / 수단
  - 정렬 기준, 날짜 1순위, 분류 2순위, 결제수단 3순위
  - 합계(금액 합계)
  - 테이블 위에 한줄씩 입력하는 form으로 작성
  - 입력시 데이터 타입 검증 수행
  - 테이블에 행 마다 기능추가
    - 신규 입력시 행 우측에 ✍️(등록) 🗙(취소) 기능 추가
    - 생성된 단일 행 우측에 ✏️(수정) 🚮(삭제) 기능 추가
    - 생성될 때 식별자 부여됨, 입력완료 후 입력컬럼 삭제, 및 조회 테이블에 추가
  - 기본값(날짜 : 오늘)(사용금액 : 필수)(분류 : '--')(사용내역 : '')(장소 : '')(수단: '현금')
- **1.2 이후 추가 개발 가능성**
  - 가계부 정렬기능 
    - (날짜기준 오름차순 내림차순 바꿔서 정렬 변경)
    - (분류기준, 카드기준 정렬)
  - 가계부 조건 검색기능 (날짜 기준으로 검색)
  - 저장 버튼으로 여러 컬럼을 한번에 수정할 수 있는 기능 고려
  - 저장 버튼으로 작성중인 모든 내용 저장 기능 고려
  - 여러 컬럼을 한번에 삭제하는 기능 고려
  - 고정지출 기능을 매달 테이블에 고정적으로 출력 시킬지 고민
### 2. 지출을 관리하는 기능 - 고정지출
#### &nbsp;&nbsp;&nbsp;&nbsp;3.1 고정 지출 등록
#### &nbsp;&nbsp;&nbsp;&nbsp;3.1 고정 지출 단일조회


- **2-1 고정지출 테이블 생성**
  - 매월 고정으로 나가는 금액을 가계부에 입력하지 않고 별도 관리
  - 지출이름 / 금액 / 날짜 / 지출방법 / 분류 / 그룹 컬럼
  - 그룹은 1(기본), 2, 3, 4, 5 로 구분
  - 그룹별 합계, 총 합계 출력
  - 입력, 조회(전체), 수정, 삭제 기능
  - 입력시 데이터 타입 검증 수행
- **2-2 이후 추가개발 가능성**
  - 그룹을 커스텀하게 관리하는 기능
### 3. 지출 그룹(분류)을 관리하는 기능, 그룹별로 월 단위 사용 된 금액을 확인 기능
#### &nbsp;&nbsp;&nbsp;&nbsp;3.1 지출 그룹 등록
#### &nbsp;&nbsp;&nbsp;&nbsp;3.2 지출 그룹 단일조회
#### &nbsp;&nbsp;&nbsp;&nbsp;3.3 지출 그룹 수정
#### &nbsp;&nbsp;&nbsp;&nbsp;3.4 지출 그룹 삭제
#### &nbsp;&nbsp;&nbsp;&nbsp;3.5 지출 그룹 목록 조회

- 3-1 분류 테이블 관리
  - 가계부에서 지출에 등록되는 분류를 정의, 분류별로 매달 한도 금액을 설정
  - 분류 / 금액 / 날짜 / 한도금액 / 이번달 사용율 
  - 분류 총합계, 이번달 사용금액 합계, 퍼센티지로 비교
  - 분류 리스트를 가계부에서 선택해서 쓰는데, 분류명이 수정된다면 가계부에도 적용 되어야 한다.
  - 분류명이 삭제된다면 가계부에는 분류정보가 '--'로 표시되어야 한다. (빈 분류로 표시됨.)
  - 
- 3-2 **이후 추가개발 가능성**
  - 한도금액 몇퍼 이상이면 색깔 바꾸게 하는 기능
### 4. 설정 기능
- 4-1 가계부의 결제 수단 셀렉트바 작성
  - 결제 수단 등록(수정)
  - 결제 수단 기본값 적용
  - 선택 없을경우 빈값을 기본값으로 사용
- 4-2 고정 지출을 가계부에 출력할지 말지 선택
  - 고정 지출 내역은 별도 컴포넌트로 테이블을 새로 만들어 출력
  - on / off 가능
- 4-3 **이후 추가개발 가능성**
  - 가계부에서 분류의 우선 순위를 지정하는 기능
  - 일반 지출에서 나오는 분류별 이번달 금액 사용 비율 출력(on/off)
### 5. 통계 처리기능 - 배치 프로세스
- 5-1 배치 프로세스
  - 통계 기능을 위해 매달 1일 배치 프로세스를 수행
  - 해당 월의 고정지출, 수단별 합계, 분류별 합계, 목표 사용금액, 사용 비율을 별도로 가공하여 저장
  - 통계 페이지에선 이렇게 처리된 월별 기준 데이터 조회 가능
- 5-2 수동으로 프로세스 진행
  - 버튼을 통해 통계로 지난달 통계를 생성(갱신) 할 수 있다.
  - 가계부의 값의 변경이 아닌, 고정지출이나 분류 통계를 위해 가공하여 저장해야 한다.
  - 기본적으로 전달 통계에 데이터를 갱신시키기 위해 사용
- 5-3 이번달 통계 자료 조회
  - 별도의 배치 프로세스 없이 이번달 통계 자료를 조회할 수 있다.
  - 금월 고정지출, 수단별 합계, 분류별 합계, 목표 사용금액, 사용 비율을 처리하여 출력
### 6. 회원 관리 기능
- 6-1 회원가입
  - id, pw, 이름, 이메일
  - pw 중복확인, 12자 이하, 첫번째 숫자로 시작하면 안됨, 숫자+영어+특수문자 필수
- 6-2 로그인
  - 세션 기반으로 로그인
  - 쿠키를 제공하여 로그인 상태 유지(7일)
- 6-3 로그아웃
  - 쿠키 삭제
- 6-4 회원정보 수정
  - 이름, 이메일, pw 수정
- 6-5 id / 비밀번호 찾기
  - 이메일 주소 입력으로 id/pw 조회
- 6-6 **이후 추가개발 가능성**
  - 주소입력 기능
  - 카카오 주소 api 연동
  - 카카오 로그인 연동
